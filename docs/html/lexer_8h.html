Detailed Description

Lexer for the compiler.

Author
Victor Antoniazzi
Date
7 August 2020

These functions are responsible for reading the input program, eat lexer_next the tokens, look without eating lexer_look and get tokens specifically by types, e.g. lexer_get_num.

◆ lexer_next()

void lexer_next ( )

Eat the next char.

Eating char is used in compilers design to mean that this current token was loaded, consumed and is no longer necessary.

Warning
After eating the token, there is no way back. You must ensure that this current token is no longer necessary.
Note
It will update the _look
See also
lexer_look() to just take a look in the token and do not eat it.
